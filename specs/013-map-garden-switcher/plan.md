# Implementation Plan: Garden Map Switcher with Harvest Indicator

**Branch**: `013-map-garden-switcher` | **Date**: 2026-03-06 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/013-map-garden-switcher/spec.md`

## Summary

Add an in-page garden selector to the Garden Map header so owners can switch between their garden plans without using the sidebar. The active garden is labelled as the harvest default. The Harvests page bed picker is updated to filter by the active garden, making the harvest-default link functional. No backend changes required — all APIs exist from feature 012.

## Implementation Details

### GardenMap.jsx — Garden selector in header

`useGarden()` already provides `gardens`, `currentGardenId`, `setCurrentGardenId`. Add to the header row:

- If `gardens.length > 1`: render a `<select>` with all gardens; selected value = `currentGardenId`; `onChange` calls `setCurrentGardenId(id)`
- Always: show the active garden name with a small "Harvest default" pill/badge beside it
- The selector replaces the plain `<h1>` title with a more informative header; existing "Garden Map" label stays above it

### Harvests.jsx — Scope bed picker to active garden

Currently fetches all beds without `gardenId`. Fix:

1. Import `useGarden` from `GardenContext`
2. Destructure `currentGardenId`
3. Pass `gardenId: currentGardenId` as a query param to `GET /api/beds`
4. Disable the bed fetch query when `currentGardenId` is null
5. React Query key: `['beds', currentGardenId]` — consistent with GardenMap

### No changes to:
- AppLayout (sidebar garden switcher stays)
- GardenContext (no changes needed)
- Backend routes/models
- Any other page
